Early termination

Early termination is a common technique used in search engine development to reduce query latency on large indexes. The idea is simple, if we were order the posting-list in decreasing significances or relevance order measured by a query-independent static rank, e.g. Google's PageRank on a webpage. Then, as we process the posting list, we can break out when we have collected enough hits before finish walking the entire list.

An important observation is that with early termination, the query latency does not grow linearly as index size grows. To return only the top N hits is very consistent with a search engine's access pattern.

The following simple code snippet illustrates how this works: 

	    int numHitsRequested = ...
	    int numHitsCollected = ...
	    int docid = ...
	    ...
	    public int nextDoc(){
		  if (docid != NO_MORE_DOCS && numHitsCollected < numHitsRequested){
		    return (docid = nextHit());	
		  }
		  else{
		    return (docid = NO_MORE_DOCS);
		  }
	    }
